## Context

当前 `backup/main.c` 的主函数是线性流程：输入目录 → 检测U盘 → 创建文件夹 → 确认 → 执行备份 → 显示报告 → 退出。任何步骤失败时，程序直接 `return 1` 退出，用户需要重新启动并重新配置。

## Goals / Non-Goals

**Goals:**
- 备份完成显示报告后，按任意键退出，让用户有时间查看结果
- 关键步骤失败时提供重试选项，而非直接终止
- 最小化代码改动，保持现有代码结构

**Non-Goals:**
- 不重构整体代码架构（不改函数签名、不改全局变量模式）
- 不引入新的第三方库
- 不改变备份核心逻辑（文件拷贝、递归遍历等）

## Decisions

1. **主循环采用 `while(1)` + 菜单模式**
   - 在 `print_final_report()` 后增加等待用户按键的逻辑，而非新增复杂的菜单系统
   - 各个关键步骤内部增加重试循环，用 `goto` 或 `while` 实现，避免改动函数签名
   - 理由：代码量小，侵入性最低，保持现有代码风格一致

2. **重试粒度选择**
   - U 盘检测失败：循环检测，用户可插入后重试
   - U 盘选择失败/创建文件夹失败：提供重试选项
   - 备份过程中出错：仅报告，不重试单个文件（已有错误计数），报告后让用户决定是否重新执行整体备份
   - 理由：单文件重试涉及增量判断，复杂度远超本次改动范围

3. **退出方式**
   - 所有退出路径改为 `printf("按 Enter 键退出..."); getchar(); return N;`
   - 理由：与 `SetConsoleOutputCP(CP_UTF8)` 控制台模式一致，无需引入 `getch()`

## Risks / Trade-offs

- [控制台输入缓冲区问题] → 使用 `getchar()` 可能与之前的 `fgets()` 残留换行符冲突，需要在等待退出前清空输入缓冲区
- [重试循环可能导致无限等待] → 每次重试提供明确的"退出"选项，用户可随时终止
- [代码行数增加但功能增量不大] → 主要是错误处理路径的改造，核心逻辑不变
